# v5: 독자 코멘트 + AI 재코멘트 시스템

## 목적

분석 노트를 수동적으로 소비하는 것이 아니라, 능동적 사고 훈련 도구로 전환한다. 읽기 상태를 추적하고, 독자가 자기 생각을 기록하면 AI가 동료 토론 톤으로 재코멘트를 달아 "생각의 근육"을 키운다.

## 결정 사항

| 항목 | 결정 | 근거 |
|------|------|------|
| 코멘트 위치 | 분석 노트 안에 직접 | 한 파일에서 분석→사고→피드백 흐름 완결. 파일 증가 방지 |
| 트리거 방식 | `/reflect` 스킬 | 명시적 워크플로우. 노트 탐색→코멘트 확인→AI 코멘트 체계적 작성 |
| AI 톤 | 동료 토론 | 동의/반박을 명확히 하고 근거 제시. "이 부분은 동의하는데, ~때문에 다른 각도도 있다" |
| 작성 시점 | 하이브리드 | Obsidian에서 미리 작성 가능 + `/reflect` 대화형으로도 작성 가능 |

## 변경 범위

### 1. 분석 노트 frontmatter

```yaml
read_status: unread    # unread | reading | read
```

### 2. 분석 노트 본문 (마지막 섹션)

```markdown
## 독자 코멘트

> [!me]
> (여기에 자유롭게 작성)

> [!ai]
> (AI 재코멘트 — /reflect가 작성)
```

### 3. `/reflect` 스킬 신규 생성

**위치**: `skills/reflect/SKILL.md`

**호출**:
- `/reflect 파일명` — 특정 노트 지정
- `/reflect` — `read_status: read`이면서 AI 코멘트 없는 노트 자동 탐색

**흐름**:
1. 분석 노트 전체 읽기 (분석 내용 + 독자 코멘트 파악)
2. 독자 코멘트 비어있음 → 대화형: "이 노트에 대해 어떤 생각이 드세요?" → 답변을 `> [!me]`에 기록
3. 독자 코멘트 있음 → 바로 AI 재코멘트 작성
4. `read_status`를 `read`로 업데이트
5. `> [!ai]` 블록에 재코멘트 작성

**AI 재코멘트 구조**:
```markdown
> [!ai]
> **동의**: (공감하는 부분 + 왜 타당한지 근거)
>
> **다른 각도**: (반박 또는 보완 관점 + 근거/사례)
>
> **확장 질문**: (이 생각을 더 밀어붙일 수 있는 후속 질문 1개)
```

### 4. analyze-source 파이프라인 변경

- 분석 노트 생성 시 frontmatter에 `read_status: unread` 추가
- 본문 마지막에 빈 `## 독자 코멘트` 섹션 추가
- 템플릿(`insights/_templates/analysis-note.md`) 업데이트
