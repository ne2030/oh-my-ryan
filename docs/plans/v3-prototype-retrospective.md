## 검증 항목별 판단

### 1. 디렉토리 구조

**판단**: 적절함.
`insights/sources/`, `insights/analysis/`, `insights/_templates/` 3개 디렉토리가 역할별로 깔끔하게 분리됨. 파일 생성/탐색에 불편한 점 없었음.

**개선 제안**: 없음.

### 2. 템플릿

**판단**: 대체로 적절하나 소소한 개선 가능.

- `analysis-note.md` 템플릿에 "라우팅 판단 기록" 섹션이 없었음 → 실제 분석 노트에서는 이 섹션을 추가함. 템플릿에 반영 필요.
- `source-note.md` 템플릿은 충분함.

**개선 제안**: analysis-note 템플릿에 `## 라우팅 판단 기록` 섹션 추가.

### 3. 라우팅

**판단**: 적절함.
v2 Section 5 기준표 "경험 공유, 사례 글 → B"와 일치. 실제 소스가 구체적 수치와 사례를 포함하고 있어 B 추천이 자연스러웠음. Override 없음.

**개선 제안**: 없음.

### 4. 렌즈

**판단**: 적절하나 조직·문화 렌즈 경계가 모호.
기술+프로세스+인지 3개 선택은 자연스러웠음. 조직·문화 렌즈를 제외했지만, 원본 마지막 단락("코드에 대한 소유권을 내려놓아야", "실험을 허용하는 조직")은 조직·문화 렌즈에 해당. 부분적 적용이 더 적절했을 수 있음.

**개선 제안**: 렌즈 선택 시 "부분 적용" 옵션 고려. 소스의 10-20%만 해당하더라도 핵심 인사이트가 있으면 렌즈를 포함.

### 5. 계층화

**판단**: 자연스러움.
원칙 3개, 사례 5개, 프레임 2개 추출. 원칙은 일반화 가능한 수준, 사례는 구체적 수치 기반, 프레임은 패러다임 전환을 잘 포착.
약간의 모호함: "도구보다 환경이 중요하다"는 원칙이지만 프레임(관점 전환)에도 해당할 수 있음.

**개선 제안**: 하나의 인사이트가 여러 계층에 걸칠 때, 주된 계층에 배치하고 다른 계층에서 참조하는 방식 고려.

### 6. 근거 확보

**판단**: B 깊이로 충분.
내부 근거 6개 수치 정리 + 채널톡 기업 배경 웹 검색 검증 + 주장 합리성 검토 포함. B 깊이의 "간단한 검증" 범위를 적절히 충족.

**개선 제안**: 없음.

### 7. 태그/링크

**판단**: 구조적으로 작동하나 검증 불충분.
- frontmatter `tags` 필드에 6개 태그 부여 → Obsidian에서 태그 검색 가능할 것
- `source_ref` 필드에 full path 위키링크 포함
- 본문 하네스 적용 제안에 full path 위키링크 포함
- 그래프 뷰 연결 검증은 Obsidian에서 직접 확인 필요 (사용자 판단 대기)
- **노트가 1건뿐이므로 태그 기반 자동 링킹은 아직 검증 불가** → 2건째 이후 검증

**개선 제안**: 2건째 분석 후 태그 겹침으로 실제 링킹이 작동하는지 확인.

### 8. 두 번째 소스 드라이런 (Kyunghun Lee — SaaSpocalypse)

#### 8a. 라우팅 판단

- **소스 성격**: 시장/트렌드 분석 + 비즈니스 사례 (Workday, UiPath, Apple, Dell, Google 등 다수 기업 사례)
- **깊이 추천**: **B** (구체적 수치가 풍부 — 시총, ARR, 보상 패키지 등. C로 갈 수도 있지만, 이미 소스 자체에 데이터가 충분하므로 B로 검증 후 부족하면 C)
- v2 라우팅 기준: "시장/트렌드 분석 → B-C" — B로 시작하는 것이 적절

#### 8b. 렌즈 선택

- **조직·문화(Org/Culture)**: 주요 렌즈. 창업자 vs 전문경영인, 이사회 의사결정, 조직 문화 변화가 핵심 내용
- **인지(Cognitive)**: "최적화 vs 재발명" 프레임, "창업자의 복귀는 향수가 아니라 생존" 관점 전환
- **기술(Technical)**: 부분 적용. UiPath의 에이전틱 자동화(Agent Builder, Maestro, ScreenPlay) 언급
- **프로세스(Process)**: 해당 없음. 구체적 워크플로우 설명이 아님

→ 1건째(기술+프로세스+인지)와 렌즈 분포가 확연히 다름. **조직·문화가 주력 렌즈**인 첫 케이스가 됨.

#### 8c. 인사이트 계층 시뮬레이션

**원칙:**
- "판이 바뀔 때는 최적화가 아니라 재발명이 필요하다"
- "재발명은 처음부터 그 자리에 있었던 사람만이 할 수 있다"

**사례:**
- Workday: 시총 800억→400억 달러 하락, 창업자 보상 패키지 1.38억 달러
- UiPath: CEO 승계 4개월 만에 창업자 복귀, ARR 17.8억 달러 도달, 주가 저점 대비 50%+ 반등
- Dell: AI 서버 주문 분기당 123억 달러

**프레임:**
- "창업자 복귀는 향수가 아니라 생존이다" — CEO 승계 플레이북의 전제(안정적 경쟁 환경)가 AI 시대에 깨짐

#### 8d. 첫 번째 소스와의 차이점

| 차원 | 1건 (Byeongchan Park) | 2건 (Kyunghun Lee) |
|------|----------------------|-------------------|
| 주요 렌즈 | 기술 + 프로세스 + 인지 | 조직·문화 + 인지 + 기술(부분) |
| 소스 성격 | 1인칭 경험 공유 | 3인칭 시장 분석 |
| 데이터 유형 | 자기 성과 수치 | 기업 재무/시장 데이터 |
| 태그 겹침 | - | `#ai-native` 없음, `#saas`, `#market`, `#agent` 추가 예상. `#agent`만 겹침 가능 |
| 인사이트 성격 | 실행 가능한 엔지니어링 패턴 | 전략적/비즈니스 관점 인사이트 |
| 하네스 적용 | 직접적 (guardrail, workflow) | 간접적 (시장 맥락 이해) |

**핵심 차이**: 1건째는 "어떻게 할 것인가(How)"에 대한 인사이트, 2건째는 "왜 해야 하는가(Why)"에 대한 인사이트. 렌즈 분포가 다르지만 프로세스 자체는 동일하게 적용 가능.
