# reflect

분석 노트의 독자 코멘트에 AI 재코멘트를 달아 "생각의 근육"을 키운다.

## Usage

```
/reflect <파일명>
/reflect
```

- 파일명 지정: 해당 분석 노트에 대해 reflect 실행
- 인자 없음: `read_status: read`이면서 `> [!ai]` 코멘트가 없는 노트를 자동 탐색

## 동작 흐름

```
1. 대상 노트 탐색
2. 분석 노트 전체 읽기
3. 독자 코멘트 확인
4. (비어있으면) 대화형 코멘트 수집
5. AI 재코멘트 작성
6. read_status 업데이트
```

## Step 1: 대상 노트 탐색

### 파일명 지정 시

`insights/analysis/` 에서 해당 파일을 찾는다. 전체 파일명 또는 부분 매칭 지원:
- `/reflect hristo` → `2026-03-20-hristo-danchev-linkedin-feed-next-gen.md` 매칭
- `/reflect 2026-03-20-hristo` → 동일

### 인자 없음

`insights/analysis/` 의 모든 분석 노트를 스캔하여:
1. `read_status: read` 이면서
2. `> [!ai]` 블록이 비어있거나 기본값(`/reflect 스킬이 작성`)인 노트를 찾는다

여러 개면 목록을 보여주고 사용자에게 선택을 요청한다.
없으면 "AI 코멘트가 필요한 노트가 없습니다. 먼저 노트를 읽고 독자 코멘트를 작성해주세요." 안내.

## Step 2: 분석 노트 전체 읽기

대상 분석 노트의 전체 내용을 읽는다. 특히:
- **원본 요약**: 소스가 무엇에 대한 것인지 파악
- **핵심 인사이트**: 원칙/사례/프레임 구조 파악
- **분석자 코멘트**: AI가 이미 작성한 분석 관점 파악
- **독자 코멘트**: `> [!me]` 블록의 내용 확인

## Step 3: 독자 코멘트 확인

### 코멘트가 있는 경우

`> [!me]` 블록에 기본값이 아닌 실제 내용이 있으면 → Step 5로 직행.

### 코멘트가 비어있는 경우

대화형 모드로 진입:

1. 분석 노트의 제목과 원본 요약을 간략히 보여준다
2. "이 노트에 대해 어떤 생각이 드세요? 동의하는 부분, 의문점, 자기 경험과 연결되는 점 등 자유롭게 말씀해주세요." 질문
3. 사용자 답변을 `> [!me]` 블록에 기록
4. 추가로 더 말하고 싶은 것이 있는지 한 번 더 질문 (선택적)
5. Step 5로 진행

## Step 4: read_status 업데이트

분석 노트 frontmatter의 `read_status`를 `read`로 업데이트한다.
(이미 `read`이면 변경 없음)

## Step 5: AI 재코멘트 작성

독자 코멘트를 읽고, 분석 노트 전체 맥락을 고려하여 동료 토론 톤으로 재코멘트를 작성한다.

### 코멘트 구조

```markdown
> [!ai]
> **동의**: [독자 코멘트 중 타당한 부분을 짚고, 왜 타당한지 분석 노트 내용이나 외부 근거로 뒷받침]
>
> **다른 각도**: [독자가 놓쳤거나 다르게 볼 수 있는 관점. 반박이 아니라 보완. 구체적 근거/사례 포함]
>
> **확장 질문**: [이 생각을 한 단계 더 밀어붙일 수 있는 후속 질문 1개. 예/아니오로 끝나지 않는 열린 질문]
```

### 작성 원칙

- **동료 토론 톤**: 가르치지 않는다. "이건 틀렸다"가 아니라 "다른 각도에서 보면"
- **근거 필수**: 동의든 반박이든 이유를 반드시 제시. 분석 노트 내용, 다른 분석 노트, 외부 사례 인용 가능
- **독자의 수준에 맞춤**: 독자 코멘트의 깊이에 비례하여 재코멘트 깊이 조절
- **연결 제안**: 독자의 생각이 다른 분석 노트의 인사이트와 연결되면 언급 (`insights/index.md` 참조)
- **확장 질문은 구체적으로**: "어떻게 생각하세요?" 같은 범용 질문 금지. 독자 코멘트의 특정 포인트를 파고드는 질문

### 분량

- 전체 3-5문단
- 독자 코멘트가 짧으면(1-2문장) 재코멘트도 짧게
- 독자 코멘트가 길고 깊으면 재코멘트도 그에 맞게

## Step 6: 저장

- `> [!ai]` 블록에 재코멘트 작성
- `read_status`가 `read`가 아니면 업데이트

완료 후 "재코멘트를 작성했습니다. 확장 질문에 대해 더 이야기하고 싶으시면 말씀해주세요." 안내.

## 반복 대화

사용자가 확장 질문에 답하거나 추가 의견을 제시하면:
- 기존 `> [!me]` 블록에 추가 내용 append
- `> [!ai]` 블록을 새로운 재코멘트로 교체 (이전 대화를 반영한 누적 코멘트)

이 반복은 사용자가 만족할 때까지 계속할 수 있다.
