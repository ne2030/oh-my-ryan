# 관련 노트 연결 규칙

## 핵심 원칙

**태그가 같다는 것만으로는 연결하지 않는다.** 의미적 관계가 있어야 한다.

## 연결 조건 (AND — 둘 다 충족해야 연결)

1. **공유 태그 1개 이상**: 최소한의 주제 겹침
2. **의미적 관계 유형 1개 이상**: 아래 표에서 해당하는 관계가 있어야 함

## 의미적 관계 유형

| 관계 유형 | 설명 | 예시 |
|-----------|------|------|
| **보완 (How↔Why)** | 하나는 "어떻게", 다른 하나는 "왜"를 다룸 | Park(How: AI 개발 방법) ↔ Lee(Why: 시장 변화) |
| **대조 (vs)** | 같은 주제에 대해 다른 입장/접근 | A(RAG 기법 중심) vs B(데이터 클렌징 중심) |
| **확장 (→)** | 하나가 다른 하나의 구체화/심화 | 원칙 글 → 그 원칙의 구현 사례 |
| **선후 (時)** | 시간축에서 전후 관계 | 2024년 예측 → 2026년 결과 |
| **적용 (⇒)** | 하나의 인사이트를 다른 맥락에 적용 | 이론 ⇒ 하네스 적용 사례 |

## 연결 불가 판정

아래 경우에는 연결하지 않는다:

- `#agent` 같은 범용 태그만 공유하고 내용적 관계가 없는 경우
- 같은 분야라는 이유만으로 연결 (예: 둘 다 AI 관련이니까)
- 연결 설명을 쓸 때 "둘 다 ~에 대해 다룬다" 이상의 관계를 설명할 수 없는 경우

## 연결 작성 형태

```markdown
## 관련 노트

- [[insights/analysis/파일명|표시명]] — [관계유형]: [1-2문장으로 구체적 관계 설명]
```

예시:
```markdown
- [[insights/analysis/2026-03-06-kyunghun-lee-saaspocalypse|Lee — SaaSpocalypse]] — 보완(How↔Why): Lee는 AI 시대에 "왜" 적응해야 하는지 시장 맥락을 제공하고, 이 글은 "어떻게" 적응하는지 엔지니어링 패턴을 다룬다.
```

## 연결 대상이 없을 때

```markdown
## 관련 노트

- 현재 연결 가능한 노트 없음. 태그: `#tag1`, `#tag2` — 향후 동일 태그 노트 추가 시 검토.
```

억지로 연결하는 것보다 "없음"이 낫다.

## 역방향 링크

새 분석 노트를 생성할 때, 연결 대상으로 선정된 기존 노트의 "관련 노트" 섹션에도 역방향 링크를 추가한다.
